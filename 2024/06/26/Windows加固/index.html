Windows 10加固

Posted on 2024-06-26 Edited on 2024-08-08 In 安全加固

本文为了方便接入安全要求较高的网络前进行查阅，请了解各项加固措施的原理和可能带来的影响，造成不便或损失概不负责。

本文并非最终版，暂未写口令策略等，本想着系统加固根据网上搜索有手就行，结果测试发现遇到很多问题，总结出来供参考，若有建议麻烦留言！将会在评论区置顶。

关闭445端口

运行——>regedit：HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NetBT\Parameters，新建QWORD（64），十六进制，名称为新值 #1，值为0

注：32位的计算机选择QWORD（32）

运行——>services.msc，将Server服务停止并禁用

注：停止该服务已知会无法使用WIreGuard协议

重启计算机

关闭139端口

更改适配器设置，右键某网卡驱动——>属性，网络，双击”Internet协议版本4“——>常规，高级——>WINS，NetBIOS设置，禁用TCP/IP上的NetBIOS

不需要重启服务

注：当存在多个网卡驱动，需要分别关闭，如安装有虚拟化、代理软件时。

关闭135端口

运行——>dcomcnfg——>组件服务——>计算机——>右键”我的电脑“——>属性：

默认属性，去掉勾选”在此计算机上启用分布式COM“

默认协议，移除“DCOM协议”中的内容

运行——>regedit：HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Rpc，新建项：Internet

重启计算机

禁止用户更改计算机名

“此电脑”右键无“属性”

运行——>gpedit.msc——>用户配置——>管理摸板——>桌面：设置 从“计算机”图标上下文菜单中删除“属性”选项” 为 已启用

禁止非管理员用户修改IP

修改网络适配器的右键属性为灰色：

运行——>gpedit.msc——>用户配置——>管理摸板——>网络——>网络连接，将以下设置为 已启用：

禁止访问LAN连接的属性

启用/禁用LAN连接的能力

为管理员启用windows 2000 网络连接设置

开启屏幕保护程序

桌面右键——>个性化——>锁屏界面——>屏幕保护程序设置

禁止未登录关机

运行——>gpedit.msc——>计算机配置——>Windows设置——>安全设置——>本地策略——>安全选项：设置 关机: 允许系统在未登录的情况下关闭 为 已禁用

禁止关机

运行——>gpedit.msc——>用户配置——>管理摸板——>“开始”菜单和任务栏：设置 删除并阻止访问“关机”、“重新启动、“睡眠”和“休眠”命令 为 已启用

启用SYN攻击保护

运行——>regedit：HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters，新建DWORD值，十进制：

启用SYN攻击保护：值名称为SynAttackProtect，推荐值为2

指定触发SYN洪水攻击保护所必须超过的TCP连接请求数阀值：值名称为TcpMaxPortsExhausted，推荐值为5

指定处于SYN_RCVD状态的TCP连接数的阈值：值名称为TcpMaxHalfOpen，推荐值为500

指定处于至少已发送一次重传的 SYN_RCVD 状态中的 TCP 连接数的阈值：值名称为TcpMaxHalfOpenRetried，推荐值为400

数据执行保护（DEP） 默认打开

”开始“菜单——>Windows 安全中心——>应用和浏览器控制——>Exploit Protection——>Exploit Protection设置

禁用大容量存储介质（USB存储设备）

运行——>gpedit.msc——>计算机配置——>管理模板——>系统——>可移动存储访问，设置 所有可移动存储类：拒绝所有权限 为 已启用

自动播放 默认关闭

”开始“菜单——>Windows 系统——>控制面板，自动播放

关闭默认共享

运行——>regedit：

HEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\lanmanserver\Parameters，新建DWORD值：

值名称为AutoShareServer，值为0

值名称为AutoShareWKs，值为0

HEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa，新建DWORD值：

值名称为restrictanonymous，值为1

需重启操作系统。
